소개

"좋은 구조는 스타일이 없어도 아름답다."
CSS와 javascript없이 웹페이지 만들기

웹 표준과 시맨틱 마크업을 중요하게 여기는 프론트엔드 개발자입니다. 작은 디테일이 사용자 경험 전체를 좌우한다는 믿음으로, 군더더기 없는 구조와 명확한 코드를 지향합니다.

HTML(HyperText Markup Language)과 CSS(Cascading Style Sheets)의 기본기를 가장 중요하게 생각하며, 보이지 않는 곳까지 정갈한 코드를 목표로 합니다.

주요 관심 분야 — 접근성(Accessibility)*, 시맨틱 HTML, 유지보수하기 쉬운 코드 구조.

* WCAG 2.1 AA 기준을 준수합니다.
최종 수정일  —  2026. 07. 01.

Q&A

 Q1 
주로 어떤 작업을 진행하나요?

정적 페이지 제작부터 컴포넌트 단위의 마크업 설계, 접근성 개선 작업까지 진행합니다.


 Q2 
협업 방식은 어떻게 되나요?

Git 기반 브랜치 전략을 따르며, 이슈 단위로 작업 내역을 commit합니다.


 Q3 
연락은 어떻게 하면 되나요?

 Q4 
시맨틱 마크업이 정확히 무엇인가요?

시맨틱 마크업이란 태그 자체의 의미에 맞게 문서를 구성하는 방식입니다. 예를 들어 목록에는 <ul>, 인용에는 <blockquote>를 사용합니다.
