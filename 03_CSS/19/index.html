HTML/CSS 핵심 개념

시맨틱 태그

<header>, <nav>, <article>처럼 의미를 담은 HTML 태그. 접근성과 SEO에 유리.

박스 모델

content · padding · border · margin으로 이루어진 크기 체계. box-sizing: border-box로 계산 기준 통일.

선택자 & 우선순위

태그 · class · id 선택자로 요소를 지정. 명시도(specificity)가 높은 규칙이 우선.

Flexbox

display: flex로 요소를 한 방향(행·열)으로 정렬·분배. 배지와 버튼 배치에 사용.

Grid

display: grid로 만드는 2차원(행+열) 레이아웃. 정보 카드 배치에 사용.

포지셔닝

static · relative · absolute · fixed · sticky로 배치 기준을 제어. 배경 장식과 컨페티에 사용.

트랜지션 & 애니메이션

transition은 상태 전환, @keyframes + animation은 반복 모션 담당. 페이지 곳곳의 움직임에 사용.

반응형 디자인

@media 쿼리와 가변 단위(%, vw, clamp)로 화면 크기에 맞춰 레이아웃 조정.

CSS 변수

:root에 값을 정의하고 var()로 사용. 이 페이지의 모든 색은 변수 두 개로 관리.

HTML/CSS 실력 테스트

HTML/CSS의 중요 개념을 확인하는 객관식 퀴즈 5문제

Q1

화면 크기에 따라 서로 다른 스타일을 적용하는 반응형 디자인의 핵심 문법은?

정답: @media. 화면 조건에 맞을 때만 스타일을 적용하는 미디어 쿼리. @import는 CSS 불러오기, @font-face는 웹폰트 등록.

Q2

박스 모델을 안쪽에서 바깥쪽 순서로 올바르게 나열한 것은?

안쪽 여백이 padding, 그 바깥 테두리가 border, 요소 사이 여백이 margin.

Q3

자식 요소들을 한 방향(행 또는 열)으로 나란히 정렬하고 싶을 때, 부모 컨테이너에 주는 속성은?

부모에 display: flex를 주면 자식들이 한 방향으로 정렬된다. 이 페이지의 배지·버튼 배치에도 사용.

Q4

계속 반복되는 움직임을 만들기 위해 @keyframes와 짝을 이뤄 쓰는 CSS 속성은?

@keyframes로 단계를 정의하고 animation으로 적용. transition은 상태 전환, transform은 이동·회전 담당.

Q5

:root에 정의한 CSS 변수 --cyan을 꺼내 쓸 때의 올바른 문법은?

CSS 변수는 var()로 꺼내 쓴다. 이 페이지의 포인트 컬러도 전부 var(--cyan), var(--yellow).
