HTML/CSS 핵심 개념

이 페이지를 만들며 활용한 HTML과 CSS의 핵심 개념

시맨틱 태그

<header>, <nav>, <article>처럼 의미를 담은 HTML 태그. 접근성과 검색엔진 최적화(SEO)에 도움을 준다.

박스 모델

content · padding · border · margin으로 이루어진 요소의 크기 체계. box-sizing: border-box로 계산 기준을 통일할 수 있다.

선택자 & 우선순위

태그·class·id·속성 선택자로 요소를 지정하며, 명시도(specificity)가 높은 규칙이 우선 적용된다.

Flexbox

display: flex로 요소를 한 방향(행 또는 열)으로 정렬·분배한다. 이 페이지의 태그, 버튼 배치에 쓰였다.

Grid

display: grid와 grid-template-columns로 2차원 레이아웃을 만든다. 정보 카드 배치에 쓰였다.

포지셔닝

static·relative·absolute·fixed·sticky로 요소의 배치 기준을 제어한다. 배경 장식과 컨페티 효과에 쓰였다.

트랜지션 & 애니메이션

transition은 상태 변화 사이를, @keyframes와 animation은 반복되는 움직임을 만든다. 이 페이지 곳곳의 모션이 여기서 나왔다.

반응형 디자인

@media 쿼리와 %, vw, clamp() 같은 가변 단위로 화면 크기에 맞춰 레이아웃을 조정한다.

CSS 변수

:root에 --cyan처럼 값을 정의하고 var()로 꺼내 쓴다. 이 페이지의 모든 색상이 변수 두 개로 관리되어, 한 줄만 바꾸면 전체 테마가 바뀐다.

HTML/CSS 실력 테스트

HTML/CSS의 중요 개념을 확인하는 객관식 퀴즈 5문제

Q1

이 페이지의 리뷰 글처럼, 독립적으로 완결된 하나의 글을 담기에 가장 알맞은 시맨틱 태그는?

<article>은 블로그 글, 뉴스 기사, 리뷰처럼 그 자체로 완결된 콘텐츠에 쓰는 시맨틱 태그다. <div>와 <span>은 아무 의미가 없어서 접근성과 SEO에 도움을 주지 못한다. 이 페이지의 본문도 <article>로 감싸져 있다.

Q2

박스 모델을 안쪽에서 바깥쪽 순서로 올바르게 나열한 것은?

콘텐츠를 감싸는 안쪽 여백이 padding, 그 바깥의 테두리가 border, 요소 사이의 바깥 여백이 margin이다. 순서를 헷갈리면 여백이 엉뚱한 곳에 생긴다.

Q3

자식 요소들을 한 방향(행 또는 열)으로 나란히 정렬하고 싶을 때, 부모 컨테이너에 주는 속성은?

부모에 display: flex를 주면 자식들이 Flexbox 아이템이 되어 한 방향으로 정렬·분배된다. 이 페이지의 어워드 배지와 방문하기 버튼이 나란히 놓인 것도 flex 덕분이다. float는 옛날 방식이라 요즘은 거의 쓰지 않는다.

Q4

계속 반복되는 움직임을 만들기 위해 @keyframes와 짝을 이뤄 쓰는 CSS 속성은?

@keyframes로 움직임의 단계를 정의하고 animation 속성으로 요소에 적용한다. transition은 호버처럼 상태가 바뀌는 순간의 전환 효과, transform은 이동·회전·확대 같은 변형 자체를 담당한다. 이 페이지의 반짝이는 별과 흐르는 티커가 모두 animation이다.

Q5

:root에 정의한 CSS 변수 --cyan을 꺼내 쓸 때의 올바른 문법은?

CSS 변수는 var() 함수로 꺼내 쓴다. 변수 이름만 쓰거나 get() 같은 함수는 동작하지 않는다. 이 페이지의 시안·노랑 포인트 컬러도 전부 var(--cyan), var(--yellow)로 쓰이고 있다.
